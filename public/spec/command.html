２．バックエンド

データベース作成

mysql -u root
>　create database blog;
>　exit

テーブル作成

php artisan migrate

テーブルリセット

php artisan migrate:fresh

※テーブルを作り直したら必ずやること！
※プルしてページが開かなかったらこのコマンドで直るかも？

ユーザーのテーブルを見る（Fortify）

php artisan tinker
>　App\Models\User::all();　//全ユーザーの全情報
>　App\Models\User::all()->first()->toArray();　//全ユーザーの全情報を配列で見る
>　App\Models\User::first();　//最初の１件
>　App\Models\User::latest()->take(5)->get();　//最新の５件
>　App\Models\User::select('name', 'email')->get();　//全ユーザーの名前とemailのみ
>　App\Models\User::count();　//全部で何件あるかを数える
>　exit

３．フロントエンド

コンポーネント作成

php artisan make:component 〇〇　//クラス付きコンポーネントの作成

作成されるファイル
・app/View/Components/〇〇.php (ロジック用クラス)
・resources/views/components/〇〇.blade.php (表示用テンプレート) 

php artisan make:component 〇〇 --view　//匿名コンポーネントの作成

作成されるファイル
・resources/views/components/〇〇.blade.php (表示用テンプレート)
